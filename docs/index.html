AI Developer Workflows · the agentic layer

adw / how it works

adw composes three actors — you, agents, and deterministic code — into repeatable developer workflows. Code owns the orchestration; agents do the fuzzy work behind one interchangeable interface; you show up only at the two ends. One machine drives five workflows, a factory router, parallel runs, and container sandboxes.

The three actors

Every node in a workflow is one of these. The color code carries through every diagram below.

Code

Deterministic & free

Orchestration, validation gates (lint · typecheck · test), and git. Fast, repeatable, zero tokens — the load-bearing layer.

run_gates() · git_ops · steps.py

Agents

Fuzzy work

Scout, plan, build, fix, review — driven headlessly. Claude Code, Codex, or opencode, assignable per role and swappable.

You

The two ends

Prompt/approve at the start and review before ship. Everything between runs without you.

The workflow — feature, the canonical run

A run is an explicit pipeline of typed nodes. Validation failures route back into the same build session so the agent keeps its context, looping until the gates pass or the fix budget runs out.

The loop is the point. When a gate fails, adw feeds the truncated failure output back to the build agent on the same session id — so it never loses context — and re-runs every gate. This repeats up to max_fix_iterations (default 3). Review always runs on a fresh session, so it reviews with unbiased eyes.

How the tool works

One command flows down through deterministic code, which hands off to agents only at the node level and collects every artifact. This is the “separate code from agents” rule made concrete.

you

adw run feature "…"

The CLI (typer) resolves the workflow and starts a run. --async pauses at gates; --race N fans out candidates.

code

Load config — adw.yaml → AdwConfig

Gates, per-role backend + model, experts, isolation, fix budget. Repo file wins over ~/.config/adw defaults.

code

Workflow — feature.py → steps.py

Plain Python control flow. Each step returns “continue” or “stop”, and skips when already done — so a paused run resumes without re-running finished agents.

agent

Agent node — AgentRunner → Adapter → env

Resolves the role to a backend + model (+ optional expert) and runs it headlessly: claude -p --output-format json, codex exec --json, or opencode run — on the host, or inside an Apple container.

← returns output text · session id · cost, saved as a transcript
code

Code node — run_gates() · git_ops

Runs each gate as a shell command (host or container) and captures pass/fail; branches, stages, commits, opens PRs via git on the host.

↺ on failure, hands the excerpt back to the build agent's session
out

State & artifacts — .adw/runs/<id>/

state.json (rewritten atomically at every step, and the hook for resume), plan.md, review.md, raw agent transcripts, and full gate logs per attempt.

Five workflows, one machine

Each workflow composes the same steps; they differ only in which agents run and where your gates sit. ↺ marks the gate loop.

feature
scout→ plan→ approve→ build→ gates ↺→ review→ ship?→ ship
bug
scout→ diagnose→ approve→ fix + test→ gates ↺→ review→ ship?→ ship
chore
build→ gates ↺→ ship?→ ship · one workhorse agent
hotfix
scout→ approve fix→ implement→ gates ↺→ ship?→ ship ASAP
cve
research→ approve→ reproduce→ mitigate→ gates ↺→ review→ ship?→ ship
cve is defensive. reproduce writes a security test that fails against the vulnerable code (proving the hole); mitigate resumes that same session to implement the protection so the test passes — leaving a permanent regression guard. Reproduction stays inside your own test suite.

The software factory — around the loop

Loops are one piece. The factory is what routes work in, isolates it, and lets it run at scale — with you at the ends.

intake

Factory router

Classifies a ticket into the right workflow (a read-only agent, with a deterministic keyword fallback), so work routes itself.

adw route "checkout 500s in prod" → hotfix

specialization

Agent experts

Reusable specialized instructions attached to a role (optionally scoped to one workflow) — the video's “surgical hotfix agent”, templated into a file.

experts/hotfix-surgeon.md → hotfix:build

human in the loop, async

Pause & resume

Gates pause instead of blocking; approve later from anywhere. Resume replays from the checkpoint — no agent re-runs.

adw run … --async adw resume <id> --approve

isolation & scale

Worktrees · racing · containers

Each run in its own git worktree; drain a queue N-at-a-time; race N candidates, first-green-wins; or run agents inside an Apple container.

adw queue process --parallel 3 adw run hotfix … --race 3
Container auth, no keychain. A Linux VM can't reach the macOS keychain, so adw forwards secrets as -e NAME from the host. Run claude setup-token once, export CLAUDE_CODE_OAUTH_TOKEN, and in-container claude -p authenticates against your Max/Pro plan — no interactive login, no API bill.
